# Extensões de encaminhamento de tags e eventos

>[!NOTE]
>
>A Adobe Experience Platform Launch foi reformulada como um conjunto de tecnologias de coleta de dados no Adobe Experience Platform. Como resultado, várias alterações de terminologia foram implementadas na documentação do produto. Consulte o seguinte [documento](../term-updates.md) para obter uma referência consolidada das alterações de terminologia.

Uma extensão é um conjunto de código empacotado que estende as funcionalidades fornecidas por tags ou pelo encaminhamento de eventos. Se você instalar uma extensão em uma biblioteca, a extensão adicionará um conjunto de eventos, condições e/ou ações exclusivas que podem ser usadas no [configurações de regra](../ui/managing-resources/rules.md).

Para obter mais detalhes sobre como as extensões funcionam nas tags e estruturas de encaminhamento de eventos, consulte o [visão geral das extensões](../ui/managing-resources/extensions/overview.md).

Para obter a documentação sobre como instalar e usar extensões disponíveis, consulte os seguintes catálogos:

* [Extensões de tags (lado do cliente)](./client/overview.md)
* [Extensões de encaminhamento de eventos (do lado do servidor)](./server/overview.md)

## Extensões de terceiros {#third-party}

Além das extensões desenvolvidas pelo Adobe, o catálogo também inclui as extensões criadas por desenvolvedores de terceiros.

O Adobe não garante essas extensões de terceiros e, portanto, é responsabilidade dos autores de extensão escolher um processo de licença e suporte para suas extensões. Muitos desenvolvedores optaram por publicar o código fonte de seus pacotes de extensão em um repositório público do GitHub, em que os usuários do podem analisá-los quando for conveniente. Consulte a [Grupo de desenvolvedores de conexões no GitHub](https://github.com/Connections-Developers) para obter mais informações.
