# Notas de versão da Adobe Experience Platform

**Data de lançamento: 28 de junho de 2019**

Novos recursos no Adobe Experience Platform:

* [[!DNL Data Science Workspace]](#dsw)
* [[!DNL Decisioning Service]](#decisioning)
* [[!DNL Query Service]](#query)

Atualizações dos recursos existentes:

* [[!DNL Experience Data Model (XDM)]](#xdm)
* [[!DNL Segmentation Service]](#segmentation)

## [!DNL Data Science Workspace] {#dsw}

Adobe Experience Platform [!DNL Data Science Workspace] é um serviço totalmente gerenciado no [!DNL Experience Platform] que permite que os cientistas de dados gerem insights de dados e conteúdo de forma contínua em soluções de Adobe e sistemas de terceiros, criando e operacionalizando Modelos de aprendizado de máquina. [!DNL Data Science Workspace] é totalmente integrado ao [!DNL Platform] e potencializa o ciclo de vida completo da ciência de dados, incluindo a exploração e a preparação de dados XDM, seguido do desenvolvimento e da operacionalização de Modelos para enriquecer automaticamente [!DNL Real-time Customer Profile] com Insights de aprendizado de máquina.

**Principais recursos**

| Recurso | Descrição |
| -----------| ---------- |
| Provisionamento e isolamento de computação | Provisionar recursos de computação dedicados necessários para permitir que os cientistas de dados executem códigos não confiáveis no [!DNL Experience Platform] de forma segura. |
| Experiência do usuário pela primeira vez | Inclui amostras prontas para uso para várias estruturas e idiomas de aprendizado de máquina, como [!DNL Python], R, PySpark e Scala [!DNL Spark]. |
| Notebooks | Ambiente personalizado para cientistas/engenheiros de dados alimentados por [!DNL Jupyter Notebooks] para que possam preparar dados, extrair recursos e desenvolver modelos ML com uma lista de bibliotecas e estruturas populares de aprendizado de máquina. |
| Exploração de dados | Acesso ininterrupto aos dados XDM assimilados no [!DNL Platform] integrado ao [!DNL Platform Data Access] SDK. |
| Visualização de dados | Capacidade de executar consultas SQL em [!DNL Jupyter Notebooks] para acelerar a preparação de dados e a engenharia de recursos. |
| pipelines de recursos | API/SDK para Scala/PySpark para implantar pipelines de engenharia de recursos para transformar dados XDM principais em schemas de recursos. |
| Criação de modelo | Modelos e tempos de execução que permitem que os cientistas de dados se concentrem no desenvolvimento de modelos sem precisar implementar o código de infraestrutura para acessar dados e recursos de computação. Você pode importar o código do modelo e operacionalizá-lo para derivar os Insights dos dados no [!DNL Platform]. |
| Gerenciamento de modelos empresariais | Suporte para modelo de dados de vários locatários para rastrear versões de modelos e configurações de hiperparâmetros associadas para fornecer base ao ecossistema de parceiros. |
| Avaliação do modelo | Avaliar e otimizar modelos de regressão e classificação em [!DNL Python], PySpark, R e Scala. |
| Implantação do modelo | Capacidade de comparar métricas e configurações de avaliação em várias execuções de experimento e publicar o modelo ideal como um serviço. |
| Pontuação em lote | Enriquecer [!DNL Real-time Customer Profile] com insights do Machine Learning ou gravá-los como conjuntos de dados de volta para [!DNL Platform] |
| Agendamento | Integrado com [!DNL Platform] Serviço de orquestração para automatizar o treinamento, a pontuação e os pipelines de recursos do Modelo com agendamentos definidos pelo usuário por meio de APIs. |

**Problemas conhecidos**

* O agendamento e os pipelines de recursos estão disponíveis no momento somente por API, com uma interface do usuário a ser adicionada em uma versão futura.

Para obter mais informações, visite o [Visão geral do Data Science Workspace](../../data-science-workspace/home.md).

## [!DNL Decisioning Service] {#decisioning}

Adobe Experience Platform [!DNL Decisioning Service] O fornece a capacidade de selecionar de forma programática e inteligente a &quot;Próxima melhor experiência&quot; de um conjunto de opções disponíveis para um determinado indivíduo, entregá-las a qualquer canal ou aplicativo e executar relatórios e análises.

Um modelo de dados avançado pré-criado permite o caso de uso da decisão da &quot;próxima melhor oferta&quot; de uma maneira agnóstica de canal.

**Principais recursos**

| Recurso | Descrição |
| -----------| ---------- |
| Repositório de objetos de negócios | Um repositório controlado por modelos de esquema JSON permite que um desenvolvedor crie, leia, atualize e exclua uma variedade de objetos comerciais. O repositório fornece APIs de consulta expressivas de propósito geral, bem como pesquisa com reconhecimento de esquema. |
| Contêineres do repositório | No repositório de objetos de negócios, um desenvolvedor pode isolar suas preocupações em torno de projetos, unidades de negócios ou organizações ou em torno dos estágios do ciclo de vida de um projeto (por exemplo, em desenvolvimento e integração, armazenamento temporário ou para uso da produção ao vivo). Esses isolamentos são chamados de contêineres de repositório. |
| Funções e permissões | Usar o [!DNL Admin Console], uma organização pode criar e gerenciar perfis para conceder acesso direcionado aos recursos por tipo, operação de acesso e contêiner. Os usuários podem ser adicionados a esses perfis de acesso e os privilégios de acesso efetivos são automaticamente calculados a partir dessas políticas. |
| Modelo de objeto de oferta pré-criada | Sem a necessidade de criar primeiro um modelo de dados, uma [!DNL Platform] o desenvolvedor pode aproveitar os esquemas e relacionamentos JSON pré-criados para criar um catálogo de ofertas, definir regras e restrições de decisão e reunir coleções de ofertas para tomada de decisão. |
| Regras de decisão com base em dados de perfil e não perfis | Uma integração estreita com a [!DNL Real-time Customer Profile] O permite que um desenvolvedor crie regras de decisão que aproveitem os dados do perfil. As decisões podem ser tomadas não apenas usando atributos de perfil, mas também com base no histórico de eventos de experiência de um perfil e com base em entidades comerciais não relacionadas a uma identidade de usuário (por exemplo, condições de tráfego, inventário de produto). Qualquer [!DNL Experience Data Model] Entidade (XDM) para a qual existe um esquema na [!DNL Schema Registry] podem ser usadas para as regras de decisão. As regras são entidades de primeira classe e podem ser reutilizadas para qualquer uma das opções e atividades de decisão. |
| Classificação e limitação | As opções de decisão que atendem a todas as condições de qualificação e outras restrições para um determinado usuário são classificadas e a melhor opção é selecionada. Restrições adicionais por usuário e limites globais podem ser usadas para limitar a exposição das opções disponíveis, permitindo a personalização com restrições de recursos e fadiga do usuário em mente. |
| [!DNL Decisioning] REST APIs | O [!DNL Decisioning Service] pode ser chamado usando uma REST API simples para obter a Próxima Melhor Oferta para um determinado indivíduo. Uma API de métricas pode ser usada para verificar valores de limite e apresentação de oferta em tempo real. |
| Transmissão de eventos de decisão para [!DNL Data Lake] e [!DNL Query Service] | O [!DNL Decisioning Service] O cria automaticamente conjuntos de dados para fazer o stream automático de todos os eventos de decisão XDM para a [!DNL Data Lake]. Os conjuntos de dados ficam disponíveis para análise e relatórios usando [!DNL Query Service]. |
| Ativação do desenvolvedor | Opt-in de autoatendimento com a documentação no Adobe I/O, incluindo tutoriais para vários tópicos. |

**Problemas conhecidos**

* O modelo de dados de oferta não é exposto por meio do [!DNL Schema Registry] e, por conseguinte, só podem ser alargadas de forma limitada. O esquema de modelo tem estruturas incorporadas para permitir o anexo de dados personalizados. No futuro, você poderá estender uma classe base de modelo XDM para definir seus próprios domínios de decisão personalizados.
* Você deve ser provisionado com o modelo de domínio Gerenciamento de ofertas e os usuários e integrações devem ser gerenciados neste contexto de produto.

## [!DNL Query Service] {#query}

[!DNL Query Service] O oferece a capacidade de usar o SQL padrão para consultar dados no Adobe Experience Platform para suportar vários casos de uso de análise e gerenciamento de dados diferentes. É uma ferramenta sem servidor que permite que você participe de qualquer conjunto de dados na [!DNL Data Lake] e capturar os resultados da consulta como um novo conjunto de dados para uso em relatórios, [!DNL Data Science Workspace]ou para ingestão em [!DNL Profile Service].

Você pode usar [!DNL Query Service] para criar ecossistemas de análise de dados, criando uma imagem dos consumidores em seus vários canais de interação. Esses canais podem incluir:

* Sistema de ponto de venda
* Web
* Dispositivo móvel
* Sistema CRM

**Principais recursos**

| Recurso | Descrição |
| -----------| ---------- |
| Editor de consultas | Use uma ferramenta baseada na Web para gravar, validar, testar e executar consultas. Ele inclui um console para obter informações detalhadas sobre a execução de queries, bem como a capacidade de visualizar resultados de query. |
| Criação do conjunto de dados | Criar conjuntos de dados em [!DNL Experience Platform] por meio da sintaxe SQL padrão. |
| Funções definidas por Adobe | Aproveite as funções de atalho para tarefas comuns, como identificar sessões ou definir atribuição. |
| conectividade com a ferramenta BI | Use os drivers PostgreSQL (Postgres) encontrados em ferramentas de BI comuns para se conectar [!DNL Query Service] para criar relatórios e visualizações. As ferramentas compatíveis incluem: [!DNL Tableau], [!DNL Power BI]e [!DNL Looker]. Encontre informações de autenticação na guia Credenciais . |
| Conectividade da ferramenta de gerenciamento de banco de dados | Connect [!DNL Aqua Data Studio] ou [!DNL DB Visualizer] para [!DNL Query Service] para exploração de dados e criação de conjuntos de dados. [!DNL Query Service] também suporta conectividade do R Studio. Encontre informações de autenticação na guia Credenciais . |
| Ferramenta de consulta de linha de comando | Conecte o PSQL para poder executar consultas a partir da linha de comando. |
| Log de consultas | Mantém um histórico de consultas executadas por [!DNL Query Service] e permite encontrar SQL anterior para edição, execução ou criação de um conjunto de dados a partir dos resultados. |
| API de agendamento de query | Agendar consultas para execução recorrente por meio dessa API. |

**Problemas conhecidos**

* [!DNL Query Editor] mostra uma amostra de 100 linhas de resultados para suas consultas. Para manter o conjunto de resultados completo, use os recursos de criação de conjunto de dados do Log de consultas.
* As versões a curto prazo adicionarão suporte a Exibições e uma interface do usuário para aplicar agendamentos a consultas.

Para obter mais informações sobre [!DNL Query Service], consulte o [documentação do produto](../../query-service/home.md).

## [!DNL Experience Data Model] (XDM) {#xdm}

A normalização e a interoperabilidade são conceitos-chave subjacentes [!DNL Experience Platform]. [!DNL Experience Data Model] O (XDM), impulsionado pelo Adobe, é um esforço para padronizar os dados de experiência do cliente e definir esquemas para o gerenciamento da experiência do cliente.

O XDM é uma especificação publicamente documentada projetada para melhorar o poder das experiências digitais. Fornece estruturas e definições comuns para qualquer aplicativo se comunicar com serviços no Adobe Experience Platform. Ao seguir os padrões XDM, todos os dados de experiência do cliente podem ser incorporados a uma representação comum, fornecendo insights de forma mais rápida e integrada. Você pode obter informações valiosas das ações do cliente, definir públicos-alvo do cliente por meio de segmentos e usar atributos do cliente para fins de personalização.

O XDM é o mecanismo que permite [!DNL Experience Cloud], viabilizada pelo Adobe Experience Platform, para fornecer a mensagem certa à pessoa certa, no canal certo, no momento certo.

A metodologia sobre a qual [!DNL Experience Platform] é construído, [!DNL XDM System] operacionaliza [!DNL Experience Data Model] esquemas a serem usados por [!DNL Experience Platform] componentes.

**Novos recursos**

| Recurso | Descrição |
| ---------- | ------------ |
| Restrições do esquema JSON | Os seguintes tipos de dados agora têm opções adicionais na interface do usuário para definir restrições: `string` - comprimento mín/máx, padrão, valor padrão, formatos (conforme definido em [Esquema JSON - rascunho 6](https://tools.ietf.org/html/draft-wright-json-schema-01)) e `double` - mín/máx, valor padrão. |
| Personalizado `$id` | Agora você pode fornecer seu próprio `$id` ao criar recursos em solicitações POST. |
| Melhorias no desempenho do Registro de Schema | Otimização da geração do schema de união e aprimoramento do armazenamento em cache do schema para melhorar muito os tempos de resposta da API. |

**Correções de erros**

* O campo identityMap foi movido para fora do contexto/perfil e para seu próprio grupo de campos de esquema para tornar a definição das identidades mais intuitiva.
* Correção de todos os esquemas existentes com base no contexto/perfil com o context/identitymap.
* Correção de uma mensagem de erro quando nenhuma versão era fornecida.
* Correção de um erro em que [!DNL Schema Registry] O estava fornecendo respostas aleatórias para chamadas de esquema de união de perfis.
* Correção de um erro em que os esquemas de união não exibiam os campos corretos em [!DNL Schema Registry].
* Correção de um erro em que os descritores de identidade ocasionalmente não podiam ser criados com namespaces válidos.
* Correção de um problema de referência se um objeto usar `properties` em vez de `allOf`.

**Problemas conhecidos**

* Não é possível estender um [!DNL Platform]grupo de campos fornecido pela adição de um campo.
* Os descritores não são excluídos quando um grupo de campos é removido da composição do schema.
* Não é possível criar um campo enum sem rótulos.

Para saber mais sobre como trabalhar com o XDM usando o [!DNL Schema Registry] API e [!DNL Schema Editor]Por favor leia o [Documentação do sistema XDM](../../xdm/home.md).

## [!DNL Segmentation Service] {#segmentation}

[!DNL Segmentation Service] O define um subconjunto específico de perfis do seu armazenamento de perfis, descrevendo os critérios para distinguir um grupo comercializável de pessoas no seu armazenamento de perfis. Os segmentos podem se basear em dados de registro (como informações demográficas) ou em eventos de séries de tempo que representem pontos de contato do cliente com sua marca.

Por exemplo, em uma campanha de email focada em tênis de corrida, você pode usar um segmento de público-alvo de todos os usuários que procuraram por tênis de corrida nos últimos 30 dias, mas não concluíram uma compra. Outro exemplo pode ser usar um segmento para direcionar o conteúdo do site, de modo que ele seja exibido somente para visitantes que pertencem a um determinado nível do seu programa de recompensas.

**Novos recursos**

| Recurso | Descrição |
| -----------| ---------- |
| Regras de tempo relativas | Agora é possível escolher janelas de tempo contínuas, como 14 dias atrás, 3 a 5 horas atrás, etc. |
| Resumos de campo XDM | Para Atributos no painel à esquerda, os resumos estão disponíveis e fornecem uma visualização dos dados subjacentes. |
| Pesquisa no painel esquerdo | Melhora dos recursos de pesquisa para a parte de segmentos do painel esquerdo. |
| Nomes amigáveis do eVar | Aprimoramento do suporte para nomes amigáveis, permitindo ver mais facilmente quais informações são capturadas em eventos e dimensões personalizadas do Adobe Analytics. |
| Suporte à política de mesclagem | Agora é possível escolher qual política de mesclagem aplicar à definição de segmento usando uma lista suspensa simples. |

**Correções de erros**

* Correção de um problema intermitente que resultava no carregamento lento dos elementos dos atributos e eventos no painel à esquerda.
* Correção de um erro que fazia com que o avaliador retornasse a resposta &quot;NaN&quot;.
* Correção de um erro em que alguns campos abriam a tela de construção de regra incorreta.

**Problemas conhecidos**

* None.

Para obter mais informações, consulte o [Visão geral do serviço de segmentação](../../segmentation/home.md).
