# Dicionário de campo XDM


O [!DNL Experience Data Model] (XDM) define um conjunto de campos padrão que podem ser usados ao compor esquemas do Adobe Experience Platform. O uso desses campos resulta em um comportamento consistente entre os serviços do [!DNL Experience Platform], pois todos os serviços que consomem XDM entenderão explicitamente as definições específicas fornecidas por esses campos.

Consulte as [noções básicas sobre a composição de esquema](composition.md) para obter mais informações sobre os componentes básicos dos esquemas, incluindo princípios e práticas recomendadas para definir esquemas para [!DNL Experience Platform].

A tabela abaixo inclui o nome do campo (nem todos os campos XDM têm namespace usando &quot;xdm&quot;) junto com seu título e descrição, tipo e formato onde necessário, bem como seu tipo XDM correspondente e os recursos (esquemas, grupos de campos de esquema e tipos de dados) nos quais o campo pode ser encontrado.

Você pode usar a funcionalidade de pesquisa no [!DNL Schema Editor] para exibir detalhes desses recursos, conforme mostrado no [tutorial do Editor de esquemas](../tutorials/create-schema-ui.md#browse).

>[!NOTE]
>
>Se um campo estiver marcado como &quot;**OBSOLETO**&quot;, significa que ele não deve mais ser usado em novas implementações. O campo continua disponível para compatibilidade com versões anteriores, mas não será mais compatível em versões futuras.
