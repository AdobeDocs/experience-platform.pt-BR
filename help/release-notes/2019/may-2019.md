## Ingestão de dados {#ingestion}

A plataforma Adobe Experience fornece um conjunto avançado de recursos para assimilar qualquer tipo e latência de dados. Você pode assimilar usando APIs em lote ou em fluxo, usando conectores criados pela Adobe, parceiros de integração de dados ou a interface do usuário da plataforma Adobe Experience.

Para obter a documentação do produto, consulte a visão geral [da ingestão de](../../ingestion/home.md)dados.

### Ingestão em lote

Use a API de ingestão em lote para assimilar qualquer tipo de dados na Adobe Experience Platform. A API de ingestão em massa fornece um mecanismo seguro e dimensionável para assimilar dados que estejam em conformidade com schemas XDM ou não-XDM.

| Recurso | Descrição |
| -----------| ---------- |
| Suporte a formato avançado | Suporta a ingestão de JSON, Parquet, CSV |
| Validação de dados | Três níveis de validação são executados nos dados: Schema, tipo de dados e restrições |
| Criptografia de dados em movimento | Protege e protege os dados enquanto são transmitidos pelas redes de dados. |
| Uploads de várias partes | Carregamentos de arquivos grandes com suporte para arquivos de 256 MB ou mais |
| Particionado automaticamente | Os dados são particionados automaticamente para melhorar a escalabilidade e otimizar o desempenho de grandes conjuntos de dados. |

**Problemas conhecidos**

Nenhum.

Para obter a documentação do produto, consulte a visão geral [da ingestão em](../../ingestion/batch-ingestion/overview.md)lote.

### Conectores

Os Conectores da plataforma Adobe Experience ajudam você a assimilar facilmente seus dados de várias fontes, permitindo que você estruture, rotule e aprimore seus dados usando os serviços da plataforma Experience. Os conectores de plataforma permitem que você se autentique em sistemas de armazenamentos baseados em nuvem e serviços CRM, defina horários para execuções de ingestão e gerencie a throughput de ingestão de dados.

| Recurso | Descrição |
| -----------| ---------- |
| Conectores de Armazenamento baseados em nuvem | Ingressar dados do armazenamento em nuvem do Microsoft Azure Blob e do Amazon S3 na Adobe Experience Platform |
| Microsoft Dynamics CRM e Salesforce CRM | Descubra todos os dados disponíveis no Microsoft Dynamics ou no Salesforce CRM. Ingresse qualquer objeto, padrão ou personalizado do Microsoft Dynamics ou do Salesforce CRM. Dados incrementais podem ser ingeridos por hora ou por dia. |  |

**Problemas conhecidos**

* Os conectores de Armazenamento baseados em nuvem só podem ser usados por meio de chamadas de API.


Para obter a documentação do produto, consulte a visão geral [](../../sources/home.md)dos conectores.

## Modelo de dados de experiência (XDM) {#xdm}

A normalização e a interoperabilidade são conceitos-chave por trás da plataforma da experiência. O Experience Data Model (XDM), desenvolvido pela Adobe, é um esforço para padronizar os dados de experiência do cliente e definir schemas para o gerenciamento da experiência do cliente.

A XDM é uma especificação publicamente documentada projetada para melhorar o poder das experiências digitais. Fornece estruturas e definições comuns para qualquer aplicativo que se comunique com os serviços na Adobe Experience Platform. Ao aderir aos padrões XDM, todos os dados de experiência do cliente podem ser incorporados a uma representação comum, fornecendo insights de forma mais rápida e integrada. Você pode obter informações importantes das ações do cliente, definir audiências do cliente por meio de segmentos e usar atributos do cliente para fins de personalização.

O XDM é o mecanismo que permite que a Experience Cloud, capacitada pela Adobe Experience Platform, forneça a mensagem certa para a pessoa certa, no canal certo, no momento exato.

A metodologia na qual a plataforma da experiência é criada, o sistema XDM opera schemas do modelo de dados da experiência para uso pelos componentes da plataforma da experiência.

| Recurso | Descrição |
| -----------| ---------- |
| Registro do Schema | O Registro de Schemas fornece uma interface de usuário (Editor de Schemas) e RESTful API da qual todos os recursos relacionados ao schema podem ser visualizados e gerenciados. O Editor de Schemas e a API de Registro de Schemas também podem ser usados para criar e gerenciar novos schemas e recursos exclusivos para cada organização do IMS. |
| Biblioteca de Schemas | A Biblioteca de Schemas contém todos os recursos de Registro de Schemas, incluindo aqueles definidos por uma organização individual de IMS e aqueles disponibilizados para todos os usuários pela Adobe, parceiros da plataforma Experience e fornecedores cujos aplicativos são integrados à plataforma Experience. |

**Problemas conhecidos**

Nenhum.

Para saber mais sobre como trabalhar com o XDM usando a API do Registro do Schema e o Editor do Schema, leia a documentação [do Sistema](../../xdm/home.md)XDM.

## Perfil do cliente em tempo real {#profile}

O Perfil é um repositório de entidade de pesquisa genérico que reúne dados em vários ativos de dados corporativos e fornece acesso a esses dados na forma de perfis de consumidores em tempo real e eventos de séries de tempo relacionados, chamados de visualização de união. A visualização unificada ajuda a informar e capacitar ações em integrações de canais, plataformas e soluções da Adobe. Além disso, o Perfil é usado para potencializar o aprendizado de máquina e o Sensei. As APIs de Perfil também podem ser usadas diretamente para aprimorar a funcionalidade de soluções de terceiros, sistemas CRM e soluções proprietárias.

| Recurso | Descrição |
| -----------| ---------- |
| perfil do cliente em tempo real | O Perfil ingere dados por meio de entradas de dados, mescla-os aos dados existentes no repositório de perfis e cria ou modifica a visualização unificada em tempo quase real. Esses dados, que evoluem no tempo com as experiências de seus clientes, podem ser usados para fazer recomendações personalizadas. |
| visualização União | Um dos principais recursos do Perfil é a capacidade de unificar dados de vários canais. O Perfil pode fornecer uma visualização unida dos indivíduos em sua base de consumidores, em conjuntos de dados e para identidades vinculadas, chamada de visualização de união. |
| Edge | O Edge Service serve como uma estrutura para a coleta de dados de baixa latência, computação conectável e ativação rápida de dados em todos os canais endereçáveis. O Edge Service facilita a personalização em tempo real. |

**Problemas conhecidos**

Nenhum.

Para obter a documentação do produto, consulte Visão geral [do](../../profile/home.md)Perfil.

## Serviço de identidade {#identity}

O Serviço de identidade resolve o desafio fundamental colocado pelas identidades desconectadas de seus clientes, causado pela natureza fragmentada de seus dados em cada um dos diferentes sistemas em jogo no ecossistema de sua organização. Isso faz através da ponte de identidades entre dispositivos e entre os vários sistemas usados por seus consumidores para se relacionar com a sua marca. Dessa forma, o Serviço de identidade facilita a compreensão completa de seus clientes e de seus comportamentos, para que você possa oferecer experiências digitais de impacto em tempo real.

| Recurso | Descrição |
| -----------| ---------- |
| Gráfico de identidade | Um gráfico de identidade é um mapa de relações entre identidades que é atualizado com a atividade do cliente em tempo quase real. O Serviço de identidade gerencia um gráfico de identidade visível somente pela sua organização e criado com base nos seus dados, conhecido como gráfico privado. O Serviço de identidade aumenta seu gráfico particular quando um registro de dados ingeridos contém mais de uma identidade, adicionando relações entre essas identidades. |

**Problemas conhecidos**

Nenhum.

Para obter a documentação do produto, consulte Visão geral [do serviço de](../../identity-service/home.md)identidade.

## Serviço de segmentação {#segmentation}

O Serviço de segmentação define um subconjunto específico de perfis da sua loja de perfis, descrevendo os critérios que distinguem um grupo de pessoas comercializáveis na sua loja de perfis. Os segmentos podem se basear em dados de registro (como informações demográficas) ou em eventos de séries cronológicas que representem pontos de contato do cliente com sua marca.

Por exemplo, em uma campanha de email chamada &quot;Você esqueceu de comprar seus tênis?&quot;, você pode desejar um segmento de audiência de todos os usuários que procuraram tênis de corrida nos últimos 30 dias, mas que não concluíram uma compra. Ou você pode querer público alvo de conteúdo da Web para que seja exibido somente para usuários com uma determinada classificação no programa de recompensas.

| Recurso | Descrição |
| -----------| ---------- |
| API de segmentação | Os serviços RESTful fornecem acesso programático para criar segmentos de clientes. |
| Construtor de segmentos | O construtor de segmentos é um espaço de trabalho robusto e intuitivo para construir segmentos de clientes a partir dos dados do Perfil. |

**Problemas conhecidos**

Nenhum.

Para obter a documentação do produto, consulte a visão geral do Serviço de [segmentação](../../segmentation/home.md)

## Governança de dados {#governance}

A plataforma Adobe Experience permite que as empresas reúnam dados de vários sistemas corporativos para melhor permitir que os profissionais de marketing identifiquem, entendam e envolvam clientes. A plataforma de experiência inclui uma infraestrutura completa de controle de dados, incluindo a DULE (Data Usage Labeling and Implementation), para garantir o uso correto dos dados dentro da plataforma e quando compartilhados entre sistemas.

O controle de dados é uma série de estratégias e tecnologias usadas para gerenciar dados de clientes e garantir a conformidade com regulamentos, restrições e políticas aplicáveis ao uso de dados. Ele desempenha um papel fundamental na plataforma da experiência em vários níveis, incluindo catalogação, linhagem de dados, rotulagem de uso de dados, políticas de acesso a dados e controle de acesso de dados para ações de marketing.

A introdução ao controle de dados exige uma compreensão completa dos regulamentos, obrigações contratuais e políticas corporativas aplicáveis aos dados do cliente. A partir daí, os dados podem ser classificados aplicando os rótulos de uso de dados apropriados, e seu uso pode ser controlado por meio da definição de políticas de uso de dados.

A estrutura DULE simplifica e simplifica o processo de categorização de dados e criação de políticas de uso de dados por meio da interface de usuário da plataforma Experience e da API do serviço de política DULE.

| Recurso | Descrição |
| -----------| ---------- |
| Entidades da Plataforma de experiência de etiquetas | Aplique e gerencie rótulos de uso de dados no nível de conexão, conjunto de dados e campo de conjunto de dados com base em rótulos classificados predefinidos. |
| Criar políticas de uso de dados | Defina e gerencie políticas de uso de dados com base na presença de rótulos e ações de marketing solicitadas. |
| Verificar violações de política | Avalie se as políticas de uso definidas são violadas ao executar ações de marketing nos dados da plataforma. |
