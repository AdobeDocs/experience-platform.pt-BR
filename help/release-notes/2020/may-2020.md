## Atualizações da interface do usuário {#ux}

A Adobe Experience Platform está lançando atualizações no domínio e na barra de cabeçalho para melhorar sua experiência e se unificar com outros aplicativos [!DNL Experience Cloud].

- É mais fácil alternar entre suas organizações ou para um aplicativo diferente
- Ajuda do usuário aprimorada, incluindo artigos em destaque e documentação relevante para o contexto no menu Ajuda
- Capacidade de fornecer feedback sobre os [!DNL Experience Platform] e os tíquetes de suporte a arquivos

A implantação da nova experiência é gradual. Você pode visualizar a experiência em [https://experience.adobe.com/platform](https://experience.adobe.com/platform).

## [!DNL Data Science Workspace] {#dsw}

[!DNL Data Science Workspace] O usa aprendizagem de máquina e inteligência artificial para liberar insights de seus dados. Integrado ao Adobe Experience Platform, [!DNL Data Science Workspace] ajuda você a fazer previsões usando seus ativos de conteúdo e dados nas soluções do Adobe. Uma das maneiras de [!DNL Data Science Workspace] conseguir isso é usando [!DNL JupyterLab]. [!DNL JupyterLab] O é uma interface do usuário baseada na Web para  [[!DNL Project Jupyter]](https://jupyter.org/) e está totalmente integrada ao Adobe Experience Platform. Ele fornece um ambiente de desenvolvimento interativo para os cientistas de dados trabalharem com [!DNL Jupyter] blocos de anotações, códigos e dados.

**Novos recursos**

| Recurso | Descrição |
|--- | ---|
| [!DNL JupyterLab Launcher] | O [!DNL JupyterLab Launcher] agora inclui inicializações para [!DNL Spark] notebooks 2.4. [!DNL Spark] Os iniciadores do notebook 2.3 agora estão marcados como obsoletos e definidos para serem removidos em uma versão subsequente. |
| [!DNL Spark] 2.4. | As novas fórmulas de Scala ([!DNL Spark]) e PySpark agora usam [!DNL Spark] 2.4. |
| Kernels | Os notebooks Scala ([!DNL Spark]) agora são criados por meio do kernel Scala. Os notebooks PySpark agora são criados por meio do kernel [!DNL Python]. O kernel [!DNL Spark] e PySpark estão obsoletos e configurados para serem removidos em uma versão subsequente. |
| Receitas | As novas receitas PySpark e [!DNL Spark] agora seguem o workflow [!DNL Docker] semelhante às receitas [!DNL Python] e R. |

Para obter mais informações sobre como migrar seus blocos de anotações e receitas para usar [!DNL Spark] 2.4, consulte o [guia de acesso a dados do notebook](../../data-science-workspace/jupyterlab/access-notebook-data.md). Para obter informações mais gerais sobre o Data Science Workspace, consulte a [documentação de visão geral](../../data-science-workspace/home.md).

A lista mais recente de casos de uso pode ser encontrada em nosso [roteiro público](https://github.com/adobe/alloy/projects/5) que é atualizado com frequência.

**Novos recursos**

| Recurso | Descrição |
|--- | ---|
| Suporte para ECID | O SDK oferece suporte à ECID pronta para uso sem bibliotecas ou informações adicionais para instalação. |
| Interface do usuário de configuração | Gerencie as configurações de ID de configuração com a nova interface do usuário de configuração de borda em [!DNL Launch], é necessário estar na lista de permissões para acessar o . |
| Grupo de campos do esquema Adobe Experience Platform Web SDK | Um grupo de campos para uso com o [!DNL Experience Platform web SDK] que abrange todos os campos compatíveis. |
| Controles de consentimento do curso | Oferece às empresas controles sobre aceitação e recusa do [!DNL Experience Platform Web SDK]. |
| Suporte a depuração do lado do cliente no novo [!DNL Experience Cloud Debugger Extension] | Consulte solicitações do [!DNL Experience Platform web SDK], bem como rastreamentos de borda para ver como os dados fluem pelo sistema. |
| Adobe Analytics | Envie dados para [!DNL Analytics] conjuntos de relatórios por meio da configuração da borda. O XDM é nivelado em dados de contexto, suporta marcação de vários conjuntos. |
| Adobe Target | Suporte para Adobe Target. Incluindo VEC, Compositor baseado em formulário, A/B, XT, Automated Personalization, MVT. |

## [!DNL Real-time Customer Profile] {#profile}

O Adobe Experience Platform permite que você conduza experiências coordenadas, consistentes e relevantes para seus clientes, independentemente de onde ou quando eles interagirem com sua marca. Com [!DNL Real-time Customer Profile], você pode ver uma visualização holística de cada cliente individual que combina dados de vários canais, incluindo dados online, offline, CRM e de terceiros. [!DNL Profile] O permite consolidar seus dados de clientes diferentes em uma visualização unificada que oferece uma conta acionável e com carimbos de data e hora de cada interação com o cliente.

**Novos recursos**

| Recurso | Descrição |
| -----------| ---------- |
| Novas métricas de exportação de perfil | Adição de métricas para tarefas de exportação de perfil, mostrando o número total de perfis exportados e o número de perfis em cada namespace. |
| Novas métricas de informações sobre observação | A API do Observability Insights agora tem as seguintes métricas para assimilação de streaming em [!DNL Profile]: Taxa de Solicitação Recebida, Taxa de Assimilação Bem-Sucedida, Tamanho de Registro Assimilar. |
| Ponto de extremidade de GET em massa | Adição do ponto de extremidade de GET em massa à API [!DNL Real-time Customer Profile] para permitir a recuperação de vários resultados em uma única chamada de API. Agora é possível fazer GET em massa de até 100 IDs para definições de segmentos, tarefas de segmentos e políticas de mesclagem. |
| Procurar perfil por identidade | Na interface [!DNL Platform], agora você pode selecionar um namespace de identidade e fornecer um valor de identidade para navegar em um perfil. |

**Correções de erros**

- None.

**Problemas conhecidos**

- Nenhum.

Para obter mais informações sobre [!DNL Real-time Customer Profile], incluindo tutoriais e práticas recomendadas para trabalhar com dados [!DNL Profile], leia a [Visão geral do perfil do cliente em tempo real](../../profile/home.md).

## Fontes {#sources}

O Adobe Experience Platform pode assimilar dados de fontes externas e, ao mesmo tempo, permite estruturar, rotular e aprimorar esses dados usando serviços [!DNL Platform]. Você pode assimilar dados de várias fontes, como aplicativos de Adobe, armazenamento baseado em nuvem, software de terceiros e seu sistema de CRM.

[!DNL Experience Platform] O fornece uma RESTful API e uma interface interativa que permite configurar conexões de origem para vários provedores de dados com facilidade. Essas conexões de origem permitem autenticar e se conectar a sistemas de armazenamento externos e serviços CRM, definir horários para execuções de assimilação e gerenciar a taxa de transferência de assimilação de dados.

**Novos recursos**

| Recurso | Descrição |
| ------- | ----------- |
| Suporte adicional para API e interface do usuário para sistemas de armazenamento em nuvem | Novos conectores de origem para [!DNL Azure File Storage]. |
| Suporte adicional à API e à interface do usuário para bancos de dados | Novos conectores de origem para [!DNL Azure Data Explorer], IBM DB2 e [!DNL Oracle DB]. |
| Adobe Audience Manager para [!DNL Experience Platform] compartilhamento de dados | O processo de provisionamento do conector [!DNL Audience Manager] foi atualizado. [!DNL Audience Manager] Os conjuntos de dados para  [!DNL Real-time Customer Profile] estão desabilitados por padrão. Você pode escolher manualmente quais conjuntos de dados promover para [!DNL Profile]. As novas configurações padrão não são retroativas e afetam apenas o provisionamento para novos conectores [!DNL Audience Manager]. Consulte mais informações no [Guia do usuário de conjuntos de dados](../../catalog/datasets/user-guide.md). |

**Problemas conhecidos**

- Nenhum.
