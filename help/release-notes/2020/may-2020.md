## Atualizações da interface do usuário {#ux}

A Adobe Experience Platform está lançando atualizações no domínio e na barra de cabeçalho para melhorar sua experiência e se unificar com outras [!DNL Experience Cloud] aplicativos.

- É mais fácil alternar entre suas organizações ou para um aplicativo diferente
- Ajuda do usuário aprimorada, incluindo artigos em destaque e documentação relevante para o contexto no menu Ajuda
- Capacidade de fornecer feedback sobre o [!DNL Experience Platform] e tíquetes de suporte a arquivos

A implantação da nova experiência é gradual. Você pode visualizar a experiência em [https://experience.adobe.com/platform](https://experience.adobe.com/platform).

## [!DNL Data Science Workspace] {#dsw}

[!DNL Data Science Workspace] O usa aprendizagem de máquina e inteligência artificial para liberar insights de seus dados. Integrado ao Adobe Experience Platform, [!DNL Data Science Workspace] O ajuda você a fazer previsões usando seu conteúdo e ativos de dados nas soluções do Adobe. Uma das maneiras [!DNL Data Science Workspace] realiza isso através do uso de [!DNL JupyterLab]. [!DNL JupyterLab] é uma interface do usuário baseada na Web para [[!DNL Project Jupyter]](https://jupyter.org/) e é totalmente integrado ao Adobe Experience Platform. Ele fornece um ambiente de desenvolvimento interativo para os cientistas de dados trabalharem com o [!DNL Jupyter] blocos de notas, código e dados.

**Novos recursos**

| Recurso | Descrição |
|--- | ---|
| [!DNL JupyterLab Launcher] | O [!DNL JupyterLab Launcher] agora inclui os primeiros passos para [!DNL Spark] 2.4 notebooks. [!DNL Spark] Os iniciadores do notebook 2.3 agora estão marcados como obsoletos e definidos para serem removidos em uma versão subsequente. |
| [!DNL Spark] 2.4. | Nova Escala ([!DNL Spark]) e as receitas do PySpark agora usam [!DNL Spark] 2.4. |
| Kernels | Escala ([!DNL Spark]) agora são criados por meio do kernel Scala. Os notebooks PySpark agora são criados por meio do [!DNL Python] Kernel. O [!DNL Spark] e o kernel do PySpark está obsoleto e definido para ser removido em uma versão subsequente. |
| Receitas | Novo PySpark e [!DNL Spark] as receitas agora seguem o [!DNL Docker] fluxo de trabalho semelhante a [!DNL Python] e R receitas. |

Para obter mais informações sobre como migrar seus blocos de anotações e receitas para usar [!DNL Spark] 2.4, consulte o [guia de acesso aos dados do notebook](../../data-science-workspace/jupyterlab/access-notebook-data.md). Para obter informações mais gerais sobre o Data Science Workspace, consulte o [documentação de visão geral](../../data-science-workspace/home.md).

## [!DNL Destinations] {#destinations}

Em [Real-time Customer Data Platform](../../rtcdp/overview.md), os destinos são integrações pré-criadas com plataformas de destino que ativam os dados para esses parceiros de forma contínua.

**[!DNL Facebook]**

A CDP em tempo real agora oferece suporte à ativação de dados para [!DNL Facebook], permitindo ativar perfis para o [!DNL Facebook] campanhas para segmentação, personalização e supressão de público-alvo com base em emails com hash.

Para obter mais informações sobre a nova funcionalidade, consulte o [Destino do facebook](../../destinations/catalog/social/facebook.md) página.

## [!DNL Real-time Customer Profile] {#profile}

O Adobe Experience Platform permite que você conduza experiências coordenadas, consistentes e relevantes para seus clientes, independentemente de onde ou quando eles interagirem com sua marca. Com [!DNL Real-time Customer Profile], você pode ver uma visualização holística de cada cliente individual que combina dados de vários canais, incluindo dados online, offline, CRM e de terceiros. [!DNL Profile] O permite consolidar seus dados de clientes diferentes em uma visualização unificada que oferece uma conta acionável e com carimbos de data e hora de cada interação com o cliente.

**Novos recursos**

| Recurso | Descrição |
| -----------| ---------- |
| Novas métricas de exportação de perfil | Adição de métricas para tarefas de exportação de perfil, mostrando o número total de perfis exportados e o número de perfis em cada namespace. |
| Novas métricas de informações sobre observação | A API do Observability Insights agora tem as seguintes métricas para assimilação de streaming no [!DNL Profile]: Taxa de Solicitação Recebida, Taxa de Assimilação Bem-Sucedida, Tamanho de Registro Assimilar. |
| Ponto de extremidade de GET em massa | Adição do ponto de extremidade de GET em massa ao [!DNL Real-time Customer Profile] API para habilitar a recuperação de vários resultados em uma única chamada de API. Agora é possível fazer GET em massa de até 100 IDs para definições de segmentos, tarefas de segmentos e políticas de mesclagem. |
| Procurar perfil por identidade | No [!DNL Platform] Agora é possível selecionar um namespace de identidade e fornecer um valor de identidade para navegar em um perfil. |

**Correções de erros**

- None.

## Fontes {#sources}

O Adobe Experience Platform pode assimilar dados de fontes externas e, ao mesmo tempo, permitir estruturar, rotular e aprimorar esses dados usando [!DNL Platform] serviços. Você pode assimilar dados de várias fontes, como aplicativos de Adobe, armazenamento baseado em nuvem, software de terceiros e seu sistema de CRM.

[!DNL Experience Platform] O fornece uma RESTful API e uma interface interativa que permite configurar conexões de origem para vários provedores de dados com facilidade. Essas conexões de origem permitem autenticar e se conectar a sistemas de armazenamento externos e serviços CRM, definir horários para execuções de assimilação e gerenciar a taxa de transferência de assimilação de dados.

**Novos recursos**

| Recurso | Descrição |
| ------- | ----------- |
| Suporte adicional para API e interface do usuário para sistemas de armazenamento em nuvem | Novos conectores de origem para [!DNL Azure File Storage]. |
| Suporte adicional à API e à interface do usuário para bancos de dados | Novos conectores de origem para [!DNL Azure Data Explorer], IBM DB2 e [!DNL Oracle DB]. |
| Adobe Audience Manager para [!DNL Experience Platform] compartilhamento de dados | O processo de provisionamento para a [!DNL Audience Manager] O conector foi atualizado. [!DNL Audience Manager] conjuntos de dados para [!DNL Real-time Customer Profile] agora estão desabilitadas por padrão. Você pode escolher manualmente quais conjuntos de dados promover para [!DNL Profile]. As novas configurações padrão não são retroativas e afetam apenas o provisionamento para novos [!DNL Audience Manager] conectores. Consulte mais informações na seção [Guia do usuário de conjuntos de dados](../../catalog/datasets/user-guide.md). |

**Problemas conhecidos**

- Nenhum.
