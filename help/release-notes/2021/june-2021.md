# Notas de versão da Adobe Experience Platform

**Data de lançamento: 30 de junho de 2021**

Atualizações dos recursos existentes na Adobe Experience Platform:

- [Perfil do cliente em tempo real](#profile)
- [Sandboxes](#sandboxes)
- [Fontes](#sources)

## Perfil do cliente em tempo real {#profile}

O Adobe Experience Platform permite gerar experiências coordenadas, consistentes e relevantes para seus clientes, independentemente de onde e quando eles interagem com sua marca. Com o Perfil do cliente em tempo real, você pode ter uma visualização integral de cada cliente individual, combinando dados de vários canais, inclusive dados online, offline, de CRM e de terceiros. [!DNL Profile] O permite consolidar os dados do cliente em uma visualização unificada, oferecendo uma conta acionável com carimbo de data e hora de cada interação com o cliente.

| Recurso | Descrição |
| ------- | ----------- |
| Atualizações do fluxo de trabalho da política de mesclagem | Ao criar e atualizar políticas de mesclagem na interface do usuário, os usuários agora podem visualizar 20 perfis de amostra com base no esquema de união. Isso permite que os usuários visualizem a aparência dos perfis do cliente antes de salvar as configurações da política de mesclagem. Para obter mais informações, consulte [guia da interface do usuário de políticas de mesclagem](../../profile/merge-policies/ui-guide.md). |
| Relatório de sobreposição de identidade | O relatório de sobreposição de identidade faz parte da API de perfil do cliente em tempo real e fornece visibilidade sobre a composição da loja de perfis. Usar o `/previewsamplestatus` ponto de extremidade, o relatório de sobreposição de identidade expõe as identidades que mais contribuem para o público-alvo endereçável. Para saber mais, visite o [visualizar exemplo de guia de endpoint da API de status](../../profile/api/preview-sample-status.md). |

Para obter mais informações sobre o Perfil do cliente em tempo real, incluindo tutoriais e práticas recomendadas para trabalhar com a [!DNL Profile] dados, comece lendo o [Visão geral do Perfil do cliente em tempo real](../../profile/home.md).

## Sandboxes {#sandboxes}

O Adobe Experience Platform foi criado para enriquecer aplicativos de experiência digital em escala global. As empresas geralmente executam vários aplicativos de experiência digital em paralelo e precisam atender ao desenvolvimento, ao teste e à implantação desses aplicativos enquanto garantem a conformidade operacional. Para atender a essa necessidade, o Experience Platform fornece sandboxes que particionam uma única instância da Platform em ambientes virtuais separados para ajudar a desenvolver aplicativos de experiência digital.

| Recurso | Descrição |
| ------- | ----------- |
| Aprimoramentos de redefinição da sandbox de produção | Agora é possível redefinir sandboxes de produção que são usadas para compartilhamento de segmento bidirecional com o Adobe Audience Manager ou o Audience Core Service. Isso pode ser feito a partir da interface ou usando o novo `validationOnly` e `ignoreWarnings` parâmetros na API. Consulte os tutoriais em [redefinição de uma sandbox na interface do](../../sandboxes/ui/user-guide.md) e [redefinição de uma sandbox na API](../../sandboxes/api/sandboxes.md) para obter mais informações. |

## Fontes {#sources}

O Adobe Experience Platform pode assimilar dados de fontes externas e, ao mesmo tempo, estruturar, rotular e aprimorar esses dados usando os serviços da plataforma. Você pode assimilar dados de várias fontes, como aplicativos Adobe, armazenamento baseado em nuvem, software de terceiros e seu sistema de CRM.

O Experience Platform fornece uma API RESTful e uma interface interativa que permite configurar conexões de origem para vários provedores de dados com facilidade. Essas conexões de origem permitem autenticar e conectar a sistemas de armazenamento externos e serviços de CRM, definir tempos para execuções de assimilação e gerenciar a taxa de transferência de assimilação de dados.

| Recurso | Descrição |
| ------- | ----------- |
| [!DNL Veeva CRM] (Beta) | Agora você pode se conectar [!DNL Veeva CRM] para Experience Platform usando o [!DNL Flow Service] API ou a interface do usuário. Consulte a [[!DNL Veeva CRM] visão geral do conector](../../sources/connectors/crm/veeva.md) para obter mais informações. |
| Suporte para monitoramento de fluxos de dados de transmissão | Agora você pode usar o espaço de trabalho da interface de fontes para monitorar as atividades de assimilação de dados de fontes de transmissão com as métricas e o status correspondentes. Veja o tutorial sobre [monitoramento de fluxos de dados de transmissão](../../sources/tutorials/ui/monitor-streaming.md) para obter mais informações. |

Para saber mais sobre fontes, consulte a [visão geral das origens](../../sources/home.md).
