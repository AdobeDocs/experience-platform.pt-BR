## Atualizações da interface do usuário {#ux}

Adobe Experience Platform is releasing updates to the domain and header bar to improve your experience and unify with other [!DNL Experience Cloud] applications.

- É mais fácil alternar entre suas organizações ou para um aplicativo diferente
- Ajuda do usuário aprimorada, incluindo artigos em destaque e documentação relevante para o contexto no menu Ajuda
- Ability to give feedback about the [!DNL Experience Platform] and file support tickets

A implantação da nova experiência é gradual. Você pode visualização a experiência em [https://experience.adobe.com/platform](https://experience.adobe.com/platform).

## [!DNL Data Science Workspace] {#dsw}

[!DNL Data Science Workspace] usa o aprendizado de máquina e a inteligência artificial para liberar insights de seus dados. Integrado ao Adobe Experience Platform, [!DNL Data Science Workspace] ajuda você a fazer previsões usando seu conteúdo e seus ativos de dados nas soluções de Adobe. Uma das maneiras de [!DNL Data Science Workspace] conseguir isso é usando [!DNL JupyterLab]. [!DNL JupyterLab] é uma interface de usuário baseada na Web para [[!DNL Project Jupyter]](https://jupyter.org/) e é totalmente integrada ao Adobe Experience Platform. Ele fornece um ambiente de desenvolvimento interativo para que os cientistas de dados trabalhem com [!DNL Jupyter] notebooks, códigos e dados.

**Novos recursos**

| Recurso | Descrição |
|--- | ---|
| [!DNL JupyterLab Launcher] | O [!DNL JupyterLab Launcher] agora inclui iniciadores para notebooks [!DNL Spark] 2.4. [!DNL Spark] 2.3 iniciadores de notebook agora são marcados como obsoletos e definidos para serem removidos em uma versão subsequente. |
| [!DNL Spark] 2.4 | As novas fórmulas Scala ([!DNL Spark]) e PySpark agora usam [!DNL Spark] 2.4. |
| Kernels | Os notebooks Scala ([!DNL Spark]) agora são criados pelo kernel Scala. Os notebooks PySpark agora são criados pelo [!DNL Python] Kernel. O kernel [!DNL Spark] e o PySpark estão obsoletos e definidos para serem removidos em uma versão subsequente. |
| Receitas | O novo PySpark e [!DNL Spark] as receitas agora seguem o fluxo de [!DNL Docker] trabalho semelhante às receitas [!DNL Python] e R. |

Para obter mais informações sobre como migrar seus notebooks e fórmulas para usar [!DNL Spark] 2.4, consulte o guia [de acesso aos dados do](../../data-science-workspace/jupyterlab/access-notebook-data.md)notebook. Para obter informações mais gerais sobre a Data Science Workspace, consulte a documentação [de](../../data-science-workspace/home.md)visão geral.

## [!DNL Experience Platform Web SDK] e [!DNL Experience Platform Edge Network] {#edge}

O [!DNL Experience Platform Web SDK] e [!DNL Experience Platform Edge Network] permitem que os usuários enviem dados para a Adobe Experience Platform e outras soluções de Adobe em tempo real para dispositivos de usuário final e navegadores. A lista mais recente de casos de uso pode ser encontrada no nosso roteiro [](https://github.com/adobe/alloy/projects/5) público, que é atualizado com frequência.

**Novos recursos**

| Recurso | Descrição |
|--- | ---|
| Suporte para ECID | O SDK oferece suporte ao ECID pronto para uso sem quaisquer bibliotecas ou informações adicionais para instalação. |
| Interface de usuário de configuração | Gerencie suas configurações de ID com a nova interface de usuário de configuração de borda em [!DNL Launch], é necessário estar na lista de permissões para acessar. |
| Mistura Adobe Experience Platform Web SDK | Uma combinação para uso com o [!DNL Experience Platform web SDK] que abrange todos os campos suportados. |
| Controles de consentimento do curso | Fornece controles de empresas sobre aceitação e não participação do [!DNL Experience Platform Web SDK]. |
| Suporte de depuração do cliente no novo [!DNL Experience Cloud Debugger Extension] | Consulte as solicitações do [!DNL Experience Platform web SDK] e os rastreamentos de borda para ver como os dados fluem pelo sistema. |
| Adobe Analytics | Envie dados para [!DNL Analytics] conjuntos de relatórios por meio da configuração de borda. O XDM é nivelado em dados de contexto, suporta marcação de vários conjuntos. |
| Adobe Target | Suporte para Adobe Target. Incluindo VEC, Compositor baseado em formulário, A/B, XT, Automated Personalization, MVT. |
| Suporte Adobe Audience Manager | Suporte para sincronizações de [!DNL Audience Manager] ID, destinos de URL e destinos de cookie. |
| Sincronização de identidade | Renomeou `setCustomersIds` para `syncIdentity` deixar mais claro. |
| Construtor de objetos XDM | Na extensão de inicialização, agora é possível criar objetos XDM como Elementos de dados. |

For more information on [!DNL Platform Web SDK] and [!DNL Edge Network], refer to the [documentation](../../edge/home.md).

## [!DNL Real-time Customer Profile] {#profile}

A Adobe Experience Platform permite que você direcione experiências coordenadas, consistentes e relevantes para seus clientes, independentemente de onde ou quando eles interagem com sua marca. Com [!DNL Real-time Customer Profile]o, você pode ver uma visualização holística de cada cliente individual que combina dados de vários canais, incluindo dados online, offline, CRM e de terceiros. [!DNL Profile] permite consolidar seus dados de clientes diferentes em uma visualização unificada, oferecendo uma conta acionável e com carimbos de data e hora de cada interação com o cliente.

**Novos recursos**

| Recurso | Descrição |
| -----------| ---------- |
| Novas métricas de Exportação de Perfil | Adicionadas métricas a trabalhos de exportação de perfil que mostram o número total de perfis exportados e o número de perfis em cada namespace. |
| Novas métricas de Insights de Observabilidade | A API Observability Insights agora tem as seguintes métricas para a inclusão de streaming em [!DNL Profile]: Taxa de Solicitação de Entrada, Taxa de Ingestão Bem-Sucedida, Tamanho de Registro Ingerido. |
| Ponto de extremidade de GET em massa | Adicionado um ponto de extremidade de GET em massa à [!DNL Real-time Customer Profile] API para permitir a recuperação de vários resultados em uma única chamada de API. Agora é possível fazer GET em massa de até 100 IDs para definições de segmentos, trabalhos de segmento e políticas de mesclagem. |
| Procurar perfil por identidade | Na [!DNL Platform] interface do usuário agora é possível selecionar uma namespace de identidade e fornecer um valor de identidade para navegar em um perfil. |

**Correções de erros**

- None.

**Problemas conhecidos**

Para obter mais informações sobre [!DNL Real-time Customer Profile], incluindo tutoriais e práticas recomendadas para trabalhar com [!DNL Profile] dados, leia a visão geral [do Perfil do cliente em tempo](../../profile/home.md)real.

## Fontes {#sources}

A Adobe Experience Platform pode assimilar dados de fontes externas, permitindo que você estruture, rotule e aprimore esses dados usando [!DNL Platform] serviços. Você pode assimilar dados de várias fontes, como aplicativos de Adobe, armazenamentos baseados em nuvem, software de terceiros e seu sistema de CRM.

[!DNL Experience Platform] fornece uma RESTful API e uma interface interativa que permite configurar conexões de origem para vários provedores de dados com facilidade. Essas conexões de origem permitem que você se autentique e se conecte a sistemas de armazenamentos externos e serviços CRM, defina horários para execuções de ingestão e gerencie a throughput de ingestão de dados.

**Novos recursos**

| Recurso | Descrição |
| ------- | ----------- |
| Suporte adicional à API e à interface do usuário para sistemas de armazenamento em nuvem | Novos conectores de origem para [!DNL Azure File Storage]. |
| Suporte adicional à API e à interface do usuário para bancos de dados | Novos conectores de origem para [!DNL Azure Data Explorer], IBM DB2 e [!DNL Oracle DB]. |
| Adobe Audience Manager para compartilhamento de [!DNL Experience Platform] dados | O processo de provisionamento do [!DNL Audience Manager] conector foi atualizado. [!DNL Audience Manager] os conjuntos de dados para agora [!DNL Real-time Customer Profile] estão desativados por padrão. Você pode escolher manualmente para quais conjuntos de dados promover [!DNL Profile]. As novas configurações padrão não são retroativas e afetam apenas o provisionamento de novos [!DNL Audience Manager] conectores. Consulte mais informações no guia [do usuário](../../catalog/datasets/user-guide.md)Conjuntos de dados. |

**Problemas conhecidos**

- None.
